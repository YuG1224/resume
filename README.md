## 自己PR

勉強会やイベント等に積極的に参加して、他のエンジニアの方と交流して情報収集をしたり、ブログ <http://blog.yug1224.com> やGitHub <https://github.com/YuG1224> で発信したり、常に最先端の技術についての情報に触れ、プロジェクトの中でどのように応用できるように行動してきました。

例えば、位置情報プラットフォームや電子書籍サービスのプロジェクトでは、コンフリクトやデグレードが頻繁に発生していたため、GitBucketを導入してGitHub-flowのプルリクエスト駆動開発を推進したり、Mochaを導入してテスト駆動開発を推進したり、開発効率やコード品質の向上のために行動し、これを実現してきました。また、開発効率だけでなくサービスの監視運用の自動化・効率化のために、RedmineやGitBucket、fluentdやMackerelの情報をSlackで共有するChatOps環境も構築しました。メールベースだったサービス監視がチャットベースになったことで、イレギュラーな事態にもリアルタイムに対応することができるようになったり、コミュニケーションが活発になり進捗管理等もスムーズにすることができました。

ほぼ全てのプロジェクトで新規サービスの立ち上げから運用に携わりました。メンバーが少ないこともあり、各チームが密接に連携し、全ての工程を自分が担当することができ、やりがいと達成感を強く感じることができました。 これまでの開発運用経験を活かして、今後もフロントエンド、バックエンドにとらわれることなく、サービス全体を俯瞰して想像しながら開発に携わっていきたいと考えております。
